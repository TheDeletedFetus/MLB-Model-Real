FEATURE TESTING METHODOLOGY

Purpose:
Identify which inputs provide measurable predictive value.

Method:
1. Compare each feature value between teams.
2. Determine which team held the feature advantage.
3. Compare feature advantage to actual game winner.
4. Record:
   - Games Tested
   - Correct Picks
   - Incorrect Picks
   - Win %
   - Edge Magnitude
   - Market Comparison
   - ROI

Classification Guidelines:

55%+
Potential Strong Signal

53%-55%
Potential Positive Signal

50%-53%
Possible Weak Signal

48%-50%
Likely Noise

Below 48%
Potential Negative Signal

LESSONS LEARNED

1. Backtesting before feature testing can hide weak signals.
2. Confidence scores are meaningless until calibrated.
3. Higher complexity does not automatically improve predictive accuracy.
4. Historical data preservation is critical.
5. Signal discovery must precede weight optimization.
6. Win percentage alone is insufficient for betting profitability.
7. Every tracked statistic should remain available for future testing.
